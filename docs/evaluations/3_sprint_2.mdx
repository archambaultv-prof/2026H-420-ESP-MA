# Sprint 2

- **Pondération** : 10%
- **Distribution de l'énoncé** : Mardi 24 mars 2026
- **Date de remise** : Correction en classe le mardi 21 avril 2026

## Objectif

Ce sprint est la continuité du sprint 1. Vous êtes à mi-parcours de votre
projet et devez faire le point sur votre avancement. Votre travail consiste à
revisiter les besoins utilisateurs identifiés lors de la soumission du projet.
Vous devez mettre à jour cette liste de besoins utilisateurs en fonction de
votre avancement et de ce qui est réaliste de faire pour la suite du projet.

Ensuite, vous choisissez parmi ces besoins utilisateurs revisités ceux qui
vous semblent les plus importants et réalistes pour le sprint 2.

## Git et GitHub

Vous continuez d'utiliser Git et GitHub pour gérer votre projet.

## Remise

Votre dépôt doit contenir un fichier `sprint2.md` à la racine, qui présente :
- Les besoins utilisateurs revisités pour le projet
- Les besoins utilisateurs que vous avez choisis pour ce sprint ainsi que leur
  critères de réussite que vous avez définis.

:::warning

Le code qui sera corrigé sera celui sur la branche `main` de votre dépôt GitHub
**à la date de remise**. Assurez-vous que votre code est à jour et fonctionnel
sur cette branche avant la date de remise.

:::
