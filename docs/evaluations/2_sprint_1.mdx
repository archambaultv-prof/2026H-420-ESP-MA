# Sprint 1

- **Pondération** : 10%
- **Distribution de l'énoncé** : Mardi 17 février 2026
- **Date de remise** : Mercredi 11 mars 2026 à 23h59

## Objectif

L'objectif du premier sprint est de coder une première version de votre projet.
Cette première version doit être **fonctionnelle**, mais elle n'a pas besoin
d'être complète.
 
Pour ce faire, choisissez parmis les besoins utilisateurs de la soumission du
projet ceux qui vous semblent les plus importants et réalistes pour une première
version. Ensuite, développez une première version de votre projet qui répond à
ces besoins.

Vous devez toutefois, dès cette première version, commencer l'intégration de
vos différentes disciplines. Donc choisissez des besoins utilisateurs qui vous
permettront de faire des choix de conception qui intègrent la deuxième
discipline.

## Git et GitHub

Vous devez utiliser Git et GitHub pour gérer votre projet. Votre code doit être
hébergé sur un dépôt GitHub. Si ce dernier est privé, vous devez inviter le
professeur comme collaborateur (nom d'utilisateur : `archambaultv-prof`).

:::info

Envoyer l'adresse de votre dépôt GitHub au professeur via MIO.

:::

## Remise

Vous n'avez pas de remise à faire pour ce sprint. Le professeur évaluera votre
projet directement sur GitHub, en se basant sur les critères d'évaluation
présentés ci-dessous.

Votre dépôt doit contenir un fichier `sprint1.md` à la racine, qui présente les
besoins utilisateurs que vous avez choisis pour ce sprint ainsi que leur
critères de réussite que vous avez définis.

:::warning

Le code qui sera corrigé sera celui sur la branche `main` de votre dépôt GitHub
**à la date de remise**. Assurez-vous que votre code est à jour et fonctionnel
sur cette branche avant la date de remise.

:::
